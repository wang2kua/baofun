# BaoFun v3 设计文档(看板主页 + 设置页 + 微光主题色 + 录音完善)

日期:2026-07-02
状态:已确认,待实现
上一版:2026-07-01-baofun-v2-design.md(哄睡/滑动音阶/动物叫声/音量,已实现)

## 背景

v2 已上线:玩耍(拍击/滑动音阶,乐音或动物叫声)、听歌(48 首贝乐虎随机)、哄睡(棕噪音/儿歌,30 分钟渐弱)、家长菜单(右上角长按 3 秒弹出的 9 项命令列表)、音量三档、护眼(黑屏+最低亮度+自动停)。宝宝现约 11 个月。

本次是一次**导航层重构** + 两个功能完善。

## 本次目标

1. **看板即主页**(最大变化):用一个正常亮度、彩色图标的看板取代"开屏直接黑屏 + 隐藏的 9 项长按菜单"。
2. **设置页**:把散落在长按菜单里的设置收进一页,并显示当前状态(不再盲切)。
3. **微光主题色**:玩耍模式触摸光晕四色可选。
4. **录音完善**:补齐试听、删除、彩蛋开关。

## 1. 导航模型(核心重构)

```
打开 App → 【看板 Dashboard】(正常亮度、深色背景、彩色大图标)
              ├─ 🎨 玩耍模式  → 进入模式(强制最低亮度 + 黑屏)
              ├─ 🎵 听歌模式  → 进入模式(黑屏)
              ├─ 🌙 哄睡模式  → 进入模式(黑屏)
              └─ ⚙️ 设置       → 设置页
   宝宝在玩耍/听歌/哄睡时:右上角长按 3 秒 → 回到看板
```

**关键行为:**
- **亮度分工**:看板恢复系统正常亮度(给家长选择用,几秒);进入任一模式强制拉到最低亮度 + 全黑;回看板恢复。护眼只作用于宝宝真正在玩的模式。
- **看板外观**:深色背景(不刺眼),但图标用清晰彩色(家长友好、好认)。这是有意的取舍——看板是给家长看的选择界面,不是给宝宝盯的。宝宝看到的永远是选完后的黑屏模式。
- **单 Activity 不变**:内部在"看板视图 ↔ 模式视图(现有 PlayView)"之间切换。
- **返回键全程屏蔽**(宝宝退不出 app);沉浸式全屏保持。
- **取代**:原来右上角长按弹出的 9 项 AlertDialog 命令列表被"看板 + 设置页"取代。右上角长按 3 秒的手势语义从"弹菜单"变为"回看板"。

## 2. 设置页(看板 ⚙️ 进入)

一页可滚动列表,每项**显示当前选中状态**(●或✓标记),而非盲切:

- 音量:小 / ●中 / 大
- 玩耍音效:●乐音 / 动物叫声
- 哄睡内容:●白噪音 / 儿歌
- 微光主题色:🟠暖橙 / 🟢柔绿 / 🔵淡蓝 / 🩷暖粉(见 3)
- 录音管理(见 4,进入二级页)
- 飞行模式提示(说明文字)
- 退出 App

所有设置持久化到 `SharedPreferences`(复用现有 `Settings` 类,扩展字段)。

## 3. 微光主题色(新)

玩耍模式下手指触摸产生的光晕颜色,四色可选:

| 主题 | 颜色 | 备注 |
|---|---|---|
| 暖橙(默认,现有) | 低饱和琥珀 | 最护眼、最接近烛光 |
| 柔绿 | 低饱和绿 | 萤火虫感 |
| 淡蓝/青 | 低饱和青蓝 | 含蓝光,睡前不建议(设置里加小注),白天可选 |
| 暖粉 | 低饱和粉 | 柔和 |

- 存 `Settings`(新增 `glowColor` 字段)。
- `PlayView` 读取当前主题色渲染光晕(现在颜色是写死的暖橙,改为可配置)。
- 保持现有的"低饱和 + 羽化边缘 + 低 alpha + 2 秒淡出"护眼特性,只换色相。

## 4. 录音完善(新)

现状:只能录/停,玩耍模式随机彩蛋播放。`VoiceRecorder` 底层已有 `listClips/playClip/deleteClip/hasClips` 能力,但没 UI 入口。

补齐"录音管理"二级页(从设置页进入):
- **录音** / **停止**(现有能力)
- **列表**:显示已录制的条数
- **试听**每条(`playClip`)
- **删除**某条(`deleteClip`)
- **彩蛋开关**:玩耍模式是否随机播放这些录音(新增 `Settings.easterEggEnabled`,默认开)

`onBabyTap` 里的彩蛋触发加一个 `settings.easterEggEnabled` 判断。

## 5. App 图标(新)

现状:无自定义图标,用系统默认。改为一个**扁平卡通汉堡🍔**:
- 矢量绘制(Android adaptive icon:前景 VectorDrawable + 背景),不依赖位图素材,任意分辨率清晰。
- 风格:扁平卡通、圆润、纯色块(上下面包胚 + 生菜/肉夹层 + 芝麻点)。
- 背景:暖橙(与护眼主题一致)。
- 配齐 adaptive icon 全套(前景/背景层),适配华为等启动器的圆形/方形/圆角遮罩。
- 由用户"你看着来"授权,风格与配色由实现方决定(扁平卡通 + 暖橙)。

## 技术要点

- **看板视图**:新增 `DashboardView`(或用一个简单的原生布局/自绘)。单 Activity 内切换 contentView:看板 ↔ PlayView。
- **亮度切换**:进入模式调 `screen.applyMinBrightnessAndImmersive()`,回看板调 `screen.restoreBrightness()` + 保持沉浸(看板也全屏,藏状态栏)。
- **状态显示**:设置页每项读 `Settings` 当前值渲染标记。
- **主题色**:`Settings.glowColor` 枚举;`PlayView` 增加 `setGlowColor(Int)`。
- **录音管理页**:列表 UI(可用 AlertDialog 列表或简单原生列表)。
- 纯逻辑(主题色枚举映射、录音列表状态)可单元测试的部分写 JVM 测试。
- Android UI/播放部分靠 assembleDebug + 真机验证。

## 护眼取舍(明确记录)

看板是正常亮度彩色 —— 比全黑略牺牲护眼纯粹性。理由:看板只在家长选模式的几秒出现,是家长在看而非宝宝盯着;宝宝真正玩的三个模式仍强制最低亮度 + 黑屏。此取舍已与用户确认接受。淡蓝主题色含蓝光,设置里标注睡前不建议。

## 测试策略

- 单元测试:主题色枚举映射、Settings 新字段默认值/持久化逻辑(纯部分)、录音彩蛋开关 gating。
- 真机验证:看板导航(选模式→进入→长按回看板)、亮度切换、四色微光、录音录/听/删/彩蛋开关。

## 不在本次范围(留待以后)

- 动物名字中文播报(需中文语音素材,成本高)

## 兼容性

- 现有模式逻辑(玩耍/听歌/哄睡/停止、计时、音量、动物叫声、滑动音阶)全部保留,只是入口从"长按菜单"改为"看板"。
- 现有 `Settings`、`ScreenController`、`PlayView`、各播放引擎复用,做增量扩展。
